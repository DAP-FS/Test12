**Question 01:** Find the maximum yield length for the following CNF CFG:

.. code-block:: text

   S ‚Üí AB
   A ‚Üí a
   B ‚Üí b

.. raw:: html

   <details>
   <summary><strong>üìò SOLUTION (Click to Expand)</strong></summary>

**Graphviz Derivation Tree (Hidden Visualization)**  

.. graphviz::

   digraph CFG1 {
       rankdir=TB;
       node [shape=circle, style=filled, fillcolor="#f9f9f9", fontname="Arial"];
       S -> A [label=" "];
       S -> B [label=" "];
       A -> a;
       B -> b;
   }

**Derivation Tree for the string** `ab`  

- Longest path length = 2 = k  
- Maximum yield length = 2^(k ‚àí 1) = 2  

.. admonition:: Concept Insight  
   :class: tip  

   The *yield length* corresponds to the number of terminals produced at the leaf level.  
   In CNF, each derivation step can at most double the number of leaves, hence the formula  
   :math:`2^{k-1}` for the maximum yield length.

**Question 03:**  Which string is accepted by PDA for :math:`a^nb^nc^n`?

A) ``aabbcc``  
B) ``aaabbbccc``  
C) ``None of them``

.. raw:: html

   <details>
   <summary><strong>‚úÖ SOLUTION (Click to Expand)</strong></summary>

.. code-block:: text

   Answer: C) None of them ‚úó

   Explanation:
   A PDA with ONE stack cannot verify THREE equal-length groups.

   aabbcc:
   - Stack after a's: [a, a, Z‚ÇÄ]
   - After matching b's: [Z‚ÇÄ]
   - How to verify 2 c's match? Stack is already empty!

   Conclusion: This language is NOT context-free. 
   It requires a Turing Machine (2+ stacks or unlimited memory).

.. raw:: html

   </details>
