.. admonition:: **Question 1:** Construct a CFG for all strings of length 2 over :math:`\Sigma = \{a, b\}`.

    .. raw:: html

        <span class="level-indicator level-basic">BASIC</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="cfg-box">
        <strong>üìã CFG Definition:</strong><br><br>
        S ‚Üí XX<br>
        X ‚Üí a | b
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Strategy:</strong>
        <ol>
        <li><strong>Goal:</strong> Generate exactly 2 symbols</li>
        <li><strong>Approach:</strong> Concatenate two variables, each producing one symbol</li>
        <li><strong>Valid strings:</strong> aa, ab, ba, bb (2 √ó 2 = 4 combinations)</li>
        </ol>
        </div>

    **Derivation for "ab":**

    .. raw:: html

        <div class="cfg-box">
        S ‚üπ XX ‚üπ aX ‚üπ ab ‚úì
        </div>

    .. raw:: html

        <div class="highlight-note">
        <strong>üí° Key Insight:</strong> Each X can independently produce 'a' or 'b', giving us 2 √ó 2 = 4 total combinations.
        </div>

        </details>

.. admonition:: **Question 2:** Construct a CFG for :math:`L = \{a^n b^n \mid n \geq 1\}`.

    .. raw:: html

        <span class="level-indicator level-basic">BASIC</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="cfg-box">
        <strong>üìã CFG Definition:</strong><br><br>
        S ‚Üí aSb | ab
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Strategy:</strong>
        <ol>
        <li><strong>Pattern:</strong> Equal number of a's followed by equal number of b's</li>
        <li><strong>Recursive step:</strong> S ‚Üí aSb adds one 'a' at left, one 'b' at right</li>
        <li><strong>Base case:</strong> S ‚Üí ab ensures at least one of each</li>
        <li><strong>Maintains equality:</strong> Each derivation step preserves n(a) = n(b)</li>
        </ol>
        </div>

    **Derivation for "aaabbb":**

    .. raw:: html

        <div class="cfg-box">
        S ‚üπ aSb ‚üπ a(aSb)b ‚üπ a(a(aSb)b)b ‚üπ a(a(ab)b)b ‚üπ aaabbb ‚úì
        </div>

    .. graphviz::

       digraph Q2 {
          rankdir=TB;
          node [shape=circle, style=filled, fillcolor="#f9f9f9", fontname="Arial"];
          S1 [label="S"];
          S1 -> a1 [label="a"];
          S1 -> S2 [label="S"];
          S1 -> b1 [label="b"];
          S2 -> a2 [label="a"];
          S2 -> ab [label="ab"];
          {rank=same; a1; S2; b1;}
       }

    .. raw:: html

        <div class="highlight-note">
        <strong>üí° Key Insight:</strong> Each recursive step adds one 'a' at the left and one 'b' at the right, maintaining equality.
        </div>

    .. raw:: html

        </details>

.. admonition:: **Question 3:** Construct a CFG for :math:`L = \{a^n b^m \mid n, m \geq 1\}`.

    .. raw:: html

        <span class="level-indicator level-basic">BASIC</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="cfg-box">
        <strong>üìã CFG Definition:</strong><br><br>
        S ‚Üí AB<br>
        A ‚Üí aA | a<br>
        B ‚Üí bB | b
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Strategy:</strong>
        <ol>
        <li><strong>Independence:</strong> Count of a's is independent of count of b's</li>
        <li><strong>Variable A:</strong> Generates any number of a's (at least 1)</li>
        <li><strong>Variable B:</strong> Generates any number of b's (at least 1)</li>
        <li><strong>Concatenation:</strong> S ‚Üí AB puts them together in sequence</li>
        </ol>
        </div>

    **Derivation for "aaabb":**

    .. raw:: html

        <div class="cfg-box">
        S ‚üπ AB ‚üπ (aA)B ‚üπ (a(aA))B ‚üπ (aaa)B ‚üπ aaa(bB) ‚üπ aaa(b(b)) ‚üπ aaabb ‚úì
        </div>

    .. graphviz::

       digraph Q3 {
          rankdir=TB;
          node [shape=circle, style=filled, fillcolor="#f9f9f9"];
          S -> A;
          S -> B;
          A -> a1;
          A -> A2 [label="A"];
          A2 -> a2;
          A2 -> A3 [label="A"];
          A3 -> a3;
          B -> b1;
          B -> B2 [label="B"];
          B2 -> b2;
          {rank=same; a1; A2; b1;}
       }

    .. raw:: html

        <div class="highlight-note">
        <strong>üí° Key Insight:</strong> Variables A and B are independent: A generates any number of a's, B generates any number of b's.
        </div>

    .. raw:: html

        </details>

.. admonition:: **Question 4:** Construct a CFG for all **even-length palindromes** over :math:`\{a, b\}`.

    .. raw:: html

        <span class="level-indicator level-intermediate">INTERMEDIATE</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="cfg-box">
        <strong>üìã CFG Definition:</strong><br><br>
        S ‚Üí aSa | bSb | Œµ
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Strategy:</strong>
        <ol>
        <li><strong>Pattern:</strong> String reads the same forwards and backwards with even length</li>
        <li><strong>Build from center:</strong> Grammar builds palindromes from center outward</li>
        <li><strong>Symmetry:</strong> S ‚Üí aSa adds matching 'a's at both ends</li>
        <li><strong>Base case:</strong> Œµ gives even-length (including empty string)</li>
        </ol>
        </div>

    **Derivation for "abba":**

    .. raw:: html

        <div class="cfg-box">
        S ‚üπ aSa ‚üπ a(bSb)a ‚üπ a(bŒµb)a ‚üπ a(bb)a ‚üπ abba ‚úì
        </div>

    .. graphviz::

       digraph Q4 {
          rankdir=TB;
          node [shape=circle, style=filled, fillcolor="#f9f9f9"];
          S1 [label="S"];
          S1 -> a1 [label="a"];
          S1 -> S2 [label="S"];
          S1 -> a2 [label="a"];
          S2 -> b1 [label="b"];
          S2 -> eps [label="Œµ"];
          S2 -> b2 [label="b"];
          eps [label="Œµ", shape=plaintext];
          {rank=same; a1; S2; a2;}
          {rank=same; b1; eps; b2;}
       }

    .. raw:: html

        <div class="highlight-note">
        <strong>üí° Key Insight:</strong> The grammar builds palindromes from the center outward, ensuring symmetry. The base case Œµ gives even-length strings.
        </div>

    .. raw:: html

        </details>

.. admonition:: **Question 5:** Construct a CFG for all **palindromes** (odd and even) over :math:`\{a, b\}`.

    .. raw:: html

        <span class="level-indicator level-intermediate">INTERMEDIATE</span>

    .. raw:: html

        <div class="steps-box">

        <ol>
        <li><strong>Even-length:</strong> S ‚Üí aSa | bSb | Œµ (from Q4)</li>
        <li><strong>Odd-length addition:</strong> Need base cases for single characters</li>
        <li><strong>S ‚Üí a:</strong> Handles odd-length palindromes ending with 'a' at center</li>
        <li><strong>S ‚Üí b:</strong> Handles odd-length palindromes ending with 'b' at center</li>
        </ol>
        </div>

    **Examples:** aba (odd), abba (even), a (single), Œµ (empty)

    **Derivation for "aba":**

    .. raw:: html

        <div class="cfg-box">
        S ‚üπ aSa ‚üπ a(b)a ‚üπ aba ‚úì
        </div>

    .. raw:: html

        <div class="highlight-note">
        <strong>üí° Key Insight:</strong> Unlike even-length palindromes, we need base cases for single characters (a, b) to handle odd-length palindromes.
        </div>

    .. raw:: html

        </details>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Why Copying (Forward) Fails:</strong>
        <ol>
        <li><strong>The Challenge:</strong> Need exact forward copy (not reverse)</li>
        <li><strong>Contrast with ww<sup>R</sup>:</strong>
            <ul>

            <li>LIFO structure can only compare in reverse order</li>

            <li>Would need FIFO queue or two stacks (= Turing Machine)</li>

Summary: CFG Patterns & Limitations
====================================

.. raw:: html

   <div class="cfg-box">
   <strong>üéØ Essential CFG Patterns:</strong><br><br>
   
   <strong>1. Equal Count (a<sup>n</sup>b<sup>n</sup>):</strong><br>
   S ‚Üí aSb | ab<br><br>
   
   <strong>2. Palindromes (ww<sup>R</sup>):</strong><br>
   S ‚Üí aSa | bSb | a | b | Œµ<br><br>
   
   <strong>3. Independent Counts (a<sup>n</sup>b<sup>m</sup>):</strong><br>
   S ‚Üí AB, A ‚Üí aA | a, B ‚Üí bB | b<br><br>
   
   <strong>4. Nested Dependencies (a<sup>n</sup>b<sup>n</sup>c<sup>m</sup>d<sup>m</sup>):</strong><br>
   S ‚Üí XY, X ‚Üí aXb | ab, Y ‚Üí cYd | cd<br><br>
   
   <strong>5. Position Constraints:</strong><br>
   S ‚Üí aXb (starts 'a', ends 'b'), X ‚Üí aX | bX | Œµ
   </div>

.. raw:: html

   <div class="highlight-note">
   <strong>‚úì CFGs CAN handle:</strong><br>
   ‚Ä¢ Two equal counts (a<sup>n</sup>b<sup>n</sup>)<br>
   ‚Ä¢ Palin
