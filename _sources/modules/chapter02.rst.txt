Level 1: Basic Stack Operations
================================

.. admonition:: **Question 1:** Design a PDA that accepts the language :math:`L = \{a^n b^n \mid n \geq 1\}`.

    .. raw:: html

        <span class="level-indicator level-basic">BASIC</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="pda-box">
        <strong>üìã PDA Definition:</strong>
        
    - **States:** :math:`Q = \{q_0, q_1, q_2\}`
    - **Input alphabet:** :math:`\Sigma = \{a, b\}`
    - **Stack alphabet:** :math:`\Gamma = \{Z_0, A\}`
    - **Start state:** :math:`q_0`
    - **Start stack symbol:** :math:`Z_0`
    - **Final state:** :math:`F = \{q_2\}`

    .. raw:: html

        </div>

    .. raw:: html

        <div class="transition-notation">
        <strong>Transition Function Œ¥:</strong><br>
        Format: (current_state, input_symbol, stack_top) ‚Üí (next_state, stack_action)
        </div>

    **Transitions:**

    .. raw:: html

        <div class="pda-box">
        <strong>1. Push Phase (Read a's):</strong><br>
        Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, AZ‚ÇÄ)  &nbsp;&nbsp; // First 'a': push A<br>
        Œ¥(q‚ÇÄ, a, A) = (q‚ÇÄ, AA)  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp; // More 'a's: push A<br><br>
        
        <strong>2. Switch Phase (First b):</strong><br>
        Œ¥(q‚ÇÄ, b, A) = (q‚ÇÅ, Œµ)  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; // Pop one A for first 'b'<br><br>
        
        <strong>3. Pop Phase (Read b's):</strong><br>
        Œ¥(q‚ÇÅ, b, A) = (q‚ÇÅ, Œµ)  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; // Pop A for each 'b'<br><br>
        
        <strong>4. Accept:</strong><br>
        Œ¥(q‚ÇÅ, Œµ, Z‚ÇÄ) = (q‚ÇÇ, Z‚ÇÄ)  &nbsp;&nbsp;&nbsp; // Stack empty ‚Üí accept
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Step-by-Step Explanation:</strong>
        <ol>
        <li><strong>Strategy:</strong> Push one symbol for each 'a', pop one for each 'b'.</li>
        <li><strong>Phase 1:</strong> While reading 'a's, push 'A' onto stack (state q‚ÇÄ).</li>
        <li><strong>Phase 2:</strong> When first 'b' arrives, switch to popping mode (q‚ÇÄ ‚Üí q‚ÇÅ).</li>
        <li><strong>Phase 3:</strong> For remaining 'b's, pop 'A' from stack (state q‚ÇÅ).</li>
        <li><strong>Acceptance:</strong> If stack is empty when input ends, accept.</li>
        </ol>
        </div>

.. admonition:: **Question 2:** Design a PDA that accepts the language :math:`L = \{w w^R \mid w \in \{a,b\}^*\}` (even-length palindromes).

    .. raw:: html

        <span class="level-indicator level-intermediate">INTERMEDIATE</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="pda-box">
        <strong>üìã PDA Definition:</strong>
        
    - **States:** :math:`Q = \{q_0, q_1, q_2\}`
    - **Input alphabet:** :math:`\Sigma = \{a, b\}`
    - **Stack alphabet:** :math:`\Gamma = \{Z_0, a, b\}`
    - **Start state:** :math:`q_0`
    - **Start stack symbol:** :math:`Z_0`
    - **Final state:** :math:`F = \{q_2\}`

    .. raw:: html

        </div>

    **Transitions:**

    .. raw:: html

        <div class="pda-box">
        <strong>1. Push First Half:</strong><br>
        Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, aZ‚ÇÄ)  &nbsp; // Push 'a'<br>
        Œ¥(q‚ÇÄ, b, Z‚ÇÄ) = (q‚ÇÄ, bZ‚ÇÄ)  &nbsp; // Push 'b'<br>
        Œ¥(q‚ÇÄ, a, a) = (q‚ÇÄ, aa)  &nbsp;&nbsp;&nbsp; // Push more 'a's<br>
        Œ¥(q‚ÇÄ, a, b) = (q‚ÇÄ, ab)  &nbsp;&nbsp;&nbsp; // Push 'a' on top of 'b'<br>
        Œ¥(q‚ÇÄ, b, a) = (q‚ÇÄ, ba)  &nbsp;&nbsp;&nbsp; // Push 'b' on top of 'a'<br>
        Œ¥(q‚ÇÄ, b, b) = (q‚ÇÄ, bb)  &nbsp;&nbsp;&nbsp; // Push more 'b's<br><br>
        
        <strong>2. Non-deterministic Switch:</strong><br>
        Œ¥(q‚ÇÄ, Œµ, a) = (q‚ÇÅ, a)  &nbsp;&nbsp;&nbsp;&nbsp; // Guess middle, keep 'a'<br>
        Œ¥(q‚ÇÄ, Œµ, b) = (q‚ÇÅ, b)  &nbsp;&nbsp;&nbsp;&nbsp; // Guess middle, keep 'b'<br><br>
        
        <strong>3. Match Second Half:</strong><br>
        Œ¥(q‚ÇÅ, a, a) = (q‚ÇÅ, Œµ)  &nbsp;&nbsp;&nbsp;&nbsp; // Match 'a' with 'a'<br>
        Œ¥(q‚ÇÅ, b, b) = (q‚ÇÅ, Œµ)  &nbsp;&nbsp;&nbsp;&nbsp; // Match 'b' with 'b'<br><br>
        
        <strong>4. Accept:</strong><br>
        Œ¥(q‚ÇÅ, Œµ, Z‚ÇÄ) = (q‚ÇÇ, Z‚ÇÄ)  &nbsp; // Empty stack ‚Üí accept
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Step-by-Step Explanation:</strong>
        <ol>
        <li><strong>Even-Length Requirement:</strong> ww<sup>R</sup> means string reads same forwards and backwards.</li>
        <li><strong>Strategy:</strong> Push first half onto stack, then match second half by popping.</li>
        <li><strong>Non-determinism:</strong> PDA must "guess" where the middle is (Œµ-transition).</li>
        <li><strong>Phase 1:</strong> Push symbols onto stack (building w).</li>
        <li><strong>Phase 2:</strong> Non-deterministically switch to matching mode.</li>
        <li><strong>Phase 3:</strong> Pop and match remaining input (verifying w<sup>R</sup>).</li>
        </ol>
        </div>

Level 2: Multiple Variable Dependencies
========================================

.. admonition:: **Question 3:** Design a PDA that accepts :math:`L = \{a^n b^m c^m \mid n, m \geq 1\}`.

    .. raw:: html

        <span class="level-indicator level-intermediate">INTERMEDIATE</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="pda-box">
        <strong>üìã PDA Definition:</strong>
        
    - **States:** :math:`Q = \{q_0, q_1, q_2, q_3\}`
    - **Input alphabet:** :math:`\Sigma = \{a, b, c\}`
    - **Stack alphabet:** :math:`\Gamma = \{Z_0, B\}`
    - **Start state:** :math:`q_0`
    - **Final state:** :math:`F = \{q_3\}`

    .. raw:: html

        </div>

    **Transitions:**

    .. raw:: html

        <div class="pda-box">
        <strong>1. Skip a's (n independent):</strong><br>
        Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, Z‚ÇÄ)  &nbsp; // Read 'a', don't push<br>
        Œ¥(q‚ÇÄ, a, B) = (q‚ÇÄ, B)  &nbsp;&nbsp;&nbsp; // Keep reading 'a's<br><br>
        
        <strong>2. Start Pushing b's (m must equal c's):</strong><br>
        Œ¥(q‚ÇÄ, b, Z‚ÇÄ) = (q‚ÇÅ, BZ‚ÇÄ)  &nbsp; // First 'b': push B<br>
        Œ¥(q‚ÇÅ, b, B) = (q‚ÇÅ, BB)  &nbsp;&nbsp;&nbsp;&nbsp; // More 'b's: push B<br><br>
        
        <strong>3. Pop for c's:</strong><br>
        Œ¥(q‚ÇÅ, c, B) = (q‚ÇÇ, Œµ)  &nbsp;&nbsp;&nbsp;&nbsp; // First 'c': pop B<br>
        Œ¥(q‚ÇÇ, c, B) = (q‚ÇÇ, Œµ)  &nbsp;&nbsp;&nbsp;&nbsp; // More 'c's: pop B<br><br>
        
        <strong>4. Accept:</strong><br>
        Œ¥(q‚ÇÇ, Œµ, Z‚ÇÄ) = (q‚ÇÉ, Z‚ÇÄ)
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Key Strategy:</strong>
        <ol>
        <li><strong>Independence:</strong> Count of 'a's is independent of 'b's and 'c's.</li>
        <li><strong>Dependency:</strong> Count of 'b's MUST equal count of 'c's.</li>
        <li><strong>Solution:</strong> Ignore 'a's (just consume), push for 'b's, pop for 'c's.</li>
        <li><strong>Why It Works:</strong> We only use the stack to track the b/c relationship!</li>
        </ol>
        </div>

        <tr><td>1</td><td>q‚ÇÄ</td><td>abbcc</td><td>Z‚ÇÄ</td><td>Read 'a' (no push)</td></tr>

        <tr><td>4</td><td>q‚ÇÅ</td><td>cc</td><td>B B Z‚ÇÄ</td><td>Push B (second 'b')</td></tr>
        <tr><td>5</td><td>q‚ÇÇ</td><td>c</td><td>B Z‚ÇÄ</td><td>Pop B (first 'c')</td></tr>

        </div>

        <div class="example-box">
        üí≠ <strong>Pattern:</strong> When one part is independent, simply consume it without stack operations. Use the stack only for dependent parts!
        </div>

    .. raw:: html

        <div class="pda-box">

    - **Stack alphabet:** :math:`\Gamma = \{Z_0, A, B\}`
    - **Start state:** :math:`q_0`

    .. raw:: html

        </div>

    .. raw:: html

        Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, AZ‚ÇÄ)  &nbsp; // First 'a': push A<br>

        <li><strong>Solution:</strong> Use stack as a "balance tracker":

            <li>If 'a' arrives and B is on top: they cancel (pop B)</li>
            <li>If 'a' arrives and A is on top: add to a-count (push A)</li>
            <li>If 'b' arrives and A is on top: they cancel (pop A)</li>
            <li>If 'b' arrives and B is on top: add to b-count (push B)</li>

        </ol>
        </div>

Summary: PDA Design Patterns
=============================

.. raw:: html

   <div class="pda-box">
   <strong>üéØ Essential PDA Patterns:</strong><br><br>
   
   <strong>1. Simple Equal Count (a<sup>n</sup>b<sup>n</sup>):</strong><br>
   ‚Ä¢ Push for first symbol, pop for second<br>
   ‚Ä¢ Accept when stack returns to Z‚ÇÄ<br><br>
   
   <strong>2. Palindromes (ww<sup>R</sup>):</strong><br>
   ‚Ä¢ Push first half, pop second half<br>
   ‚Ä¢ Use non-determinism to guess middle<br><br>
   
   <strong>3. Independent Parts (a<sup>n</sup>b<sup>m</sup>c<sup>m</sup>):</strong><br>
   ‚Ä¢ Ignore independent symbols (don't push)<br>
   ‚Ä¢ Use stack only for dependent parts<br><br>
   
   <strong>4. Any-Order Equal Counts:</strong><br>
   ‚Ä¢ Push A for 'a', push B for 'b'<br>
   ‚Ä¢ Pop opposite symbol when possible (cancellation)<br><br>
   
   <strong>5. Sequential Pairs (a<sup>n</sup>b<sup>n</sup>c<sup>m</sup>d<sup>m</sup>):</strong><br>
   ‚Ä¢ Handle first pair completely<br>
   ‚Ä¢ Then handle second pair<br>
   ‚Ä¢ Use different stack symbols
   </div>

.. raw:: html

   <div class="highlight-note">
   <strong>‚ö° PDA Limitations:</strong><br>
   ‚úì Can handle: two counts, palindromes, balanced parentheses<br>
   ‚úó Cannot handle: three equal counts (a<sup>n</sup>b<sup>n</sup>c<sup>n</sup>), copying (ww), context-sensitive patterns
   </div>
