.. admonition:: **Question 1:** Design a PDA for :math:`L = \{a^n b^n \mid n \geq 1\}`.

    .. raw:: html

        <span class="level-indicator level-basic">BASIC</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="pda-box">
        <strong>üìã PDA Definition:</strong>
        
    - **States:** :math:`Q = \{q_0, q_1, q_2\}`
    - **Input:** :math:`\Sigma = \{a, b\}`
    - **Stack:** :math:`\Gamma = \{Z_0, A\}`
    - **Start:** :math:`q_0`, **Final:** :math:`F = \{q_2\}`

    .. raw:: html

        </div>

    **Transitions:**

    .. raw:: html

        <div class="pda-box">
        Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, AZ‚ÇÄ) // First 'a'<br>
        Œ¥(q‚ÇÄ, a, A) = (q‚ÇÄ, AA) // More 'a's<br>
        Œ¥(q‚ÇÄ, b, A) = (q‚ÇÅ, Œµ) // First 'b', switch<br>
        Œ¥(q‚ÇÅ, b, A) = (q‚ÇÅ, Œµ) // More 'b's<br>
        Œ¥(q‚ÇÅ, Œµ, Z‚ÇÄ) = (q‚ÇÇ, Z‚ÇÄ) // Accept
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Strategy:</strong>
        <ol>
        <li>Push A for each 'a' read</li>
        <li>Pop A for each 'b' read</li>
        <li>Accept if stack empties exactly when input ends</li>
        </ol>
        </div>

    .. raw:: html

        <div class="accepted">
        Example: "aaabbb" is ACCEPTED
        </div>

    .. raw:: html

        </details>

.. admonition:: **Question 2:** Design a PDA for :math:`L = \{ww^R \mid w \in \{a,b\}^*\}` (even palindromes).

    .. raw:: html

        <span class="level-indicator level-intermediate">INTERMEDIATE</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="pda-box">
        <strong>üìã PDA Definition:</strong>
        
    - **States:** :math:`Q = \{q_0, q_1, q_2\}`
    - **Input:** :math:`\Sigma = \{a, b\}`
    - **Stack:** :math:`\Gamma = \{Z_0, a, b\}`

    .. raw:: html

        </div>

    **Transitions:**

    .. raw:: html

        <div class="pda-box">
        <strong>Push Phase:</strong><br>
        Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, aZ‚ÇÄ)<br>
        Œ¥(q‚ÇÄ, b, Z‚ÇÄ) = (q‚ÇÄ, bZ‚ÇÄ)<br>
        Œ¥(q‚ÇÄ, a, a) = (q‚ÇÄ, aa)<br>
        Œ¥(q‚ÇÄ, a, b) = (q‚ÇÄ, ab)<br>
        Œ¥(q‚ÇÄ, b, a) = (q‚ÇÄ, ba)<br>
        Œ¥(q‚ÇÄ, b, b) = (q‚ÇÄ, bb)<br><br>
        
        <strong>Guess Middle (Œµ-transition):</strong><br>
        Œ¥(q‚ÇÄ, Œµ, a) = (q‚ÇÅ, a)<br>
        Œ¥(q‚ÇÄ, Œµ, b) = (q‚ÇÅ, b)<br><br>
        
        <strong>Match Phase:</strong><br>
        Œ¥(q‚ÇÅ, a, a) = (q‚ÇÅ, Œµ)<br>
        Œ¥(q‚ÇÅ, b, b) = (q‚ÇÅ, Œµ)<br>
        Œ¥(q‚ÇÅ, Œµ, Z‚ÇÄ) = (q‚ÇÇ, Z‚ÇÄ)
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Strategy:</strong>
        <ol>
        <li>Push first half onto stack</li>
        <li>Non-deterministically guess middle point</li>
        <li>Match second half by popping and comparing</li>
        </ol>
        </div>

    .. raw:: html

        <div class="accepted">
        Example: "abba" is ACCEPTED
        </div>

    .. raw:: html

        </details>

.. admonition:: **Question 3:** Design a PDA for :math:`L = \{a^n b^m c^m \mid n,m \geq 1\}`.

    .. raw:: html

        <span class="level-indicator level-intermediate">INTERMEDIATE</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="pda-box">
        Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, Z‚ÇÄ) // Skip a's<br>
        Œ¥(q‚ÇÄ, a, B) = (q‚ÇÄ, B)<br>
        Œ¥(q‚ÇÄ, b, Z‚ÇÄ) = (q‚ÇÅ, BZ‚ÇÄ) // Push for b's<br>
        Œ¥(q‚ÇÅ, b, B) = (q‚ÇÅ, BB)<br>
        Œ¥(q‚ÇÅ, c, B) = (q‚ÇÇ, Œµ) // Pop for c's<br>
        Œ¥(q‚ÇÇ, c, B) = (q‚ÇÇ, Œµ)<br>
        Œ¥(q‚ÇÇ, Œµ, Z‚ÇÄ) = (q‚ÇÉ, Z‚ÇÄ)
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Key Insight:</strong>
        <ul>
        <li>'a' count is independent - just consume</li>
        <li>'b' and 'c' must be equal - use stack</li>
        </ul>
        </div>

    .. raw:: html

        <div class="accepted">
        Example: "aabbcc" is ACCEPTED
        </div>

    .. raw:: html

        </details>

.. admonition:: **Question 6:** Design a PDA for :math:`L = \{w \mid N_a(w) = N_b(w), w \in \{a,b\}^*\}`.

    .. raw:: html

        <span class="level-indicator level-advanced">ADVANCED</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="pda-box">
        Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, AZ‚ÇÄ) // First 'a'<br>
        Œ¥(q‚ÇÄ, a, A) = (q‚ÇÄ, AA) // More 'a's<br>
        Œ¥(q‚ÇÄ, a, B) = (q‚ÇÄ, Œµ) // Cancel with 'b'<br>
        Œ¥(q‚ÇÄ, b, Z‚ÇÄ) = (q‚ÇÄ, BZ‚ÇÄ) // First 'b'<br>
        Œ¥(q‚ÇÄ, b, B) = (q‚ÇÄ, BB) // More 'b's<br>
        Œ¥(q‚ÇÄ, b, A) = (q‚ÇÄ, Œµ) // Cancel with 'a'<br>
        Œ¥(q‚ÇÄ, Œµ, Z‚ÇÄ) = (q‚ÇÅ, Z‚ÇÄ) // Accept
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Cancellation Strategy:</strong>
        <ul>
        <li>If 'a' arrives and B on top: cancel (pop B)</li>
        <li>If 'a' arrives and A on top: add to count (push A)</li>
        <li>If 'b' arrives and A on top: cancel (pop A)</li>
        <li>If 'b' arrives and B on top: add to count (push B)</li>
        <li>Accept when stack returns to Z‚ÇÄ</li>
        </ul>
        </div>

    .. raw:: html

        <div class="accepted">
        Example: "abba" is ACCEPTED
        </div>

    .. raw:: html

        </details>

        </div>

.. admonition:: **Question 8:** Design a PDA for :math:`L = \{a^i b^j c^k \mid i=j \text{ or } j=k\}`.

    .. raw:: html

        <span class="level-indicator level-advanced">ADVANCED</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="pda-box">
        <strong>Non-deterministically choose:</strong><br><br>
        
        <strong>Path 1 (i=j):</strong><br>
        Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÅ, AZ‚ÇÄ)<br>

.. admonition:: **Question 15:** Prove that :math:`L = \{a^n b^n c^n \mid n \geq 1\}` is NOT context-free.

    .. raw:: html

        <span class="level-indicator level-advanced">ADVANCED</span>

    **Solution:**

    .. raw:: html

        <details>
        <summary><b>üîç Show Complete Solution</b></summary>

    .. raw:: html

        <div class="not-possible">
        NOT POSSIBLE with a PDA
        </div>

    .. raw:: html

        <div class="steps-box">
        <strong>üéØ Proof Using Pumping Lemma:</strong>
        <ol>
        <li><strong>The Problem:</strong> Need to enforce three equal counts simultaneously</li>
        <li><strong>PDA Limitation:</strong> A PDA has only ONE stack</li>
        <li><strong>What PDAs Can Do:</strong> Match TWO counts (like a<sup>n</sup>b<sup>n</sup>)</li>
        <li><strong>Why Three Fails:</strong>
            <ul>
            <li>To match a's and b's, we push 'a's and pop for 'b's</li>
            <li>But then we can't also verify c's match without a second stack</li>
            <li>Would need 2D memory (two stacks or equivalent)</li>
            </ul>
        </li>
        <li><strong>Formal Proof:</strong> Use Pumping Lemma for CFLs:
            <ul>
            <li>Let p be pumping length, consider w = a<sup>p</sup>b<sup>p</sup>c<sup>p</sup></li>
            <li>Any decomposition w = uvxyz with |vxy| ‚â§ p</li>
            <li>vxy spans at most 2 symbol types</li>
            <li>Pumping uv¬≤xy¬≤z breaks three-way equality</li>
            </ul>
        </li>
        <li><strong>Language Class:</strong> This is Context-Sensitive (needs LBA)</li>
        </ol>
        </div>

    .. raw:: html

        <div class="example-box">
        üí≠ <strong>Key Insight:</strong> A PDA can "count" one relationship using its stack. To count two independent relationships simultaneously, you need more power than a PDA provides.
        </div>

    .. raw:: html

        </details>

Summary: What Makes Languages Non-Context-Free
===============================================

.. raw:: html

   <div class="pda-box">
   <strong>üéØ Patterns That CANNOT Be Handled by PDAs:</strong><br><br>
   
   <strong>1. Three or More Equal Counts:</strong><br>
   ‚Ä¢ a<sup>n</sup>b<sup>n</sup>c<sup>n</sup> - needs to track three relationships<br>
   ‚Ä¢ Requires more than one stack<br><br>
   
   <strong>2. Forward Copying:</strong><br>
   ‚Ä¢ ww - needs to remember and compare in same direction<br>
   ‚Ä¢ w<sup>n</sup> - multiple forward copies<br>
   ‚Ä¢ Stack is LIFO, can only compare in reverse<br><br>
   
   <strong>3. Arithmetic Properties:</strong><br>
   ‚Ä¢ Primes: a<sup>n</sup> where n is prime<br>
   ‚Ä¢ Perfect squares: a<sup>n¬≤</sup><br>
   ‚Ä¢ Factorials: a<sup>n!</sup><br>
   ‚Ä¢ Exponentials: a<sup>n^m</sup><br>
   ‚Ä¢ Requires divisibility testing or complex arithmetic<br><br>
   
   <strong>4. Crossed Dependencies:</strong><br>
   ‚Ä¢ a<sup>i</sup>b<sup>j</sup>c<sup>k</sup>d<sup>l</sup> where i=k AND j=l<br>
   ‚Ä¢ Two independent equalities that "cross" each other<br><br>
   
   <strong>5. Mixed Constraints:</strong><br>
   ‚Ä¢ www<sup>R</sup> - combines copying with reverse<br>
   ‚Ä¢ Multiple simultaneous conditions
   </div>

.. raw:: html

   <div class="highlight-note">
   <strong>‚ö° Why These Fail:</strong><br><br>
   
   <strong>PDA Limitations:</strong><br>
   ‚Ä¢ Only ONE stack (LIFO structure)<br>
   ‚Ä¢ Can track at most ONE relationship<br>
   ‚Ä¢ Can compare only in REVERSE order<br>
   ‚Ä¢ Cannot perform arithmetic beyond counting<br><br>
   
   <strong>What's Needed:</strong><br>
   ‚Ä¢ Two stacks ‚Üí Linear Bounded Automaton (LBA)<br>
   ‚Ä¢ Full tape ‚Üí Turing Machine<br>
   ‚Ä¢ Multiple simultaneous counters ‚Üí Beyond CFLs
   </div>

.. raw:: html

   <div class="steps-box">
   <strong>üéì Proof Techniques:</strong><br>
   <ol>
   <li><strong>Pumping Lemma for CFLs:</strong> Show no matter how you pump, language membership breaks</li>
   <li><strong>Closure Properties:</strong> Show if L were CFL, then intersecting with regular language gives known non-CFL</li>
   <li><strong>Reduction:</strong> Show if L were CFL, then another known non-CFL would also be CFL (contradiction)</li>
   <li><strong>Direct Argument:</strong> Show inherent limitation of PDA (like LIFO for ww)</li>
   </ol>
   </div>

Summary: PDA Design Patterns
=============================

.. raw:: html

   <div class="pda-box">
   <strong>üéØ Essential Patterns:</strong><br><br>
   
   <strong>1. Equal Count (a<sup>n</sup>b<sup>n</sup>):</strong><br>
   Push for first, pop for second. Most fundamental pattern.<br><br>
   
   <strong>2. Palindromes (ww<sup>R</sup>):</strong><br>
   Push first half, guess middle, match second half.<br><br>
   
   <strong>3. Independent Parts (a<sup>n</sup>b<sup>m</sup>c<sup>m</sup>):</strong><br>
   Ignore independent symbols, track only dependent ones.<br><br>
   
   <strong>4. Any-Order Equal (N<sub>a</sub>=N<sub>b</sub>):</strong><br>
   Cancellation pattern - push/pop opposite symbols.<br><br>
   
   <strong>5. Ratio Patterns (a<sup>2n</sup>b<sup>n</sup>):</strong><br>
   Use state alternation or multiple push/pop.<br><br>
   
   <strong>6. Sequential Pairs (a<sup>n</sup>b<sup>n</sup>c<sup>m</sup>d<sup>m</sup>):</strong><br>
   Handle completely in sequence, different stack symbols.<br><br>
   
   <strong>7. Union (L‚ÇÅ ‚à™ L‚ÇÇ):</strong><br>
   Non-deterministically choose which language to recognize.
   </div>
