  • Start state: \(q_0\)

Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, aZ‚ÇÄ)   // Push 'a'
Œ¥(q‚ÇÄ, b, Z‚ÇÄ) = (q‚ÇÄ, bZ‚ÇÄ)   // Push 'b'

Œ¥(q‚ÇÄ, b, a) = (q‚ÇÄ, ba)     // Push 'b' on top of 'a'
Œ¥(q‚ÇÄ, b, b) = (q‚ÇÄ, bb)     // Push more 'b's

2. Non-deterministic Switch:
Œ¥(q‚ÇÄ, Œµ, a) = (q‚ÇÅ, a)      // Guess middle, keep 'a'
Œ¥(q‚ÇÄ, Œµ, b) = (q‚ÇÅ, b)      // Guess middle, keep 'b'

Œ¥(q‚ÇÅ, a, a) = (q‚ÇÅ, Œµ)      // Match 'a' with 'a'
Œ¥(q‚ÇÅ, b, b) = (q‚ÇÅ, Œµ)      // Match 'b' with 'b'

Œ¥(q‚ÇÅ, Œµ, Z‚ÇÄ) = (q‚ÇÇ, Z‚ÇÄ)   // Empty stack ‚Üí accept
üéØ Step-by-Step Explanation:
  1. Even-Length Requirement: wwR means string reads same forwards and backwards.
  2. Strategy: Push first half onto stack, then match second half by popping.
  3. Non-determinism: PDA must "guess" where the middle is (Œµ-transition).
  4. Phase 1: Push symbols onto stack (building w).
  5. Phase 2: Non-deterministically switch to matching mode.
  6. Phase 3: Pop and match remaining input (verifying wR).

Question 3: Design a PDA that accepts \(L = \{a^n b^m c^m \mid n, m \geq 1\}\).

INTERMEDIATE

Solution:

üìã PDA Definition:

  • States: \(Q = \{q_0, q_1, q_2, q_3\}\)

  • Input alphabet: \(\Sigma = \{a, b, c\}\)

  • Stack alphabet: \(\Gamma = \{Z_0, B\}\)

  • Start state: \(q_0\)

  • Final state: \(F = \{q_3\}\)

Transitions:

1. Skip a's (n independent):
Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, Z‚ÇÄ)   // Read 'a', don't push
Œ¥(q‚ÇÄ, a, B) = (q‚ÇÄ, B)     // Keep reading 'a's

2. Start Pushing b's (m must equal c's):
Œ¥(q‚ÇÄ, b, Z‚ÇÄ) = (q‚ÇÅ, BZ‚ÇÄ)   // First 'b': push B
Œ¥(q‚ÇÅ, b, B) = (q‚ÇÅ, BB)      // More 'b's: push B

3. Pop for c's:
Œ¥(q‚ÇÅ, c, B) = (q‚ÇÇ, Œµ)      // First 'c': pop B
Œ¥(q‚ÇÇ, c, B) = (q‚ÇÇ, Œµ)      // More 'c's: pop B

4. Accept:
Œ¥(q‚ÇÇ, Œµ, Z‚ÇÄ) = (q‚ÇÉ, Z‚ÇÄ)
üéØ Key Strategy:
  1. Independence: Count of 'a's is independent of 'b's and 'c's.
  2. Dependency: Count of 'b's MUST equal count of 'c's.
  3. Solution: Ignore 'a's (just consume), push for 'b's, pop for 'c's.
  4. Why It Works: We only use the stack to track the b/c relationship!

Question 5: Design a PDA for \(L = \{a^n b^n c^m d^m \mid n, m \geq 1\}\).

ADVANCED

Solution:

üîç Show Complete Solution
üìã PDA Definition:

  • States: \(Q = \{q_0, q_1, q_2, q_3, q_4\}\)

  • Input alphabet: \(\Sigma = \{a, b, c, d\}\)

  • Stack alphabet: \(\Gamma = \{Z_0, A, C\}\)

  • Start state: \(q_0\)

  • Final state: \(F = \{q_4\}\)

Transitions:

1. First Pair (anbn):
Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, AZ‚ÇÄ)   // Push A for 'a'
Œ¥(q‚ÇÄ, a, A) = (q‚ÇÄ, AA)
Œ¥(q‚ÇÄ, b, A) = (q‚ÇÅ, Œµ)      // Pop A for 'b'
Œ¥(q‚ÇÅ, b, A) = (q‚ÇÅ, Œµ)

2. Second Pair (cmdm):
Œ¥(q‚ÇÅ, c, Z‚ÇÄ) = (q‚ÇÇ, CZ‚ÇÄ)   // Push C for 'c'
Œ¥(q‚ÇÇ, c, C) = (q‚ÇÇ, CC)
Œ¥(q‚ÇÇ, d, C) = (q‚ÇÉ, Œµ)      // Pop C for 'd'
Œ¥(q‚ÇÉ, d, C) = (q‚ÇÉ, Œµ)

3. Accept:
Œ¥(q‚ÇÉ, Œµ, Z‚ÇÄ) = (q‚ÇÑ, Z‚ÇÄ)
üéØ Key Strategy:
  1. Two Independent Pairs: (a,b) and (c,d) are separate equal-count pairs.
  2. Sequential Processing: First handle anbn, then handle cmdm.
  3. Stack Reuse: After first pair is balanced (stack returns to Z‚ÇÄ), we use stack again for second pair.
  4. Different Stack Symbols: Use A for first pair, C for second pair to avoid confusion.

Question 6: Design a PDA for \(L = \{w \mid N_a(w) = N_b(w), w \in \{a,b\}^*\}\) (equal a‚Äôs and b‚Äôs in any order).

ADVANCED

Solution:

üîç Show Complete Solution
üìã PDA Definition:

  • States: \(Q = \{q_0, q_1\}\)

  • Input alphabet: \(\Sigma = \{a, b\}\)

  • Stack alphabet: \(\Gamma = \{Z_0, A, B\}\)

  • Start state: \(q_0\)

  • Final state: \(F = \{q_1\}\)

Transitions:

1. Handle 'a' (multiple cases):
Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, AZ‚ÇÄ)   // First 'a': push A
Œ¥(q‚ÇÄ, a, A) = (q‚ÇÄ, AA)     // More 'a's: push A
Œ¥(q‚ÇÄ, a, B) = (q‚ÇÄ, Œµ)      // 'a' cancels 'b': pop B

2. Handle 'b' (multiple cases):
Œ¥(q‚ÇÄ, b, Z‚ÇÄ) = (q‚ÇÄ, BZ‚ÇÄ)   // First 'b': push B
Œ¥(q‚ÇÄ, b, B) = (q‚ÇÄ, BB)     // More 'b's: push B
Œ¥(q‚ÇÄ, b, A) = (q‚ÇÄ, Œµ)      // 'b' cancels 'a': pop A

3. Accept:
Œ¥(q‚ÇÄ, Œµ, Z‚ÇÄ) = (q‚ÇÅ, Z‚ÇÄ)   // Balanced ‚Üí accept
üéØ Key Strategy:
  1. Challenge: a's and b's can be interleaved in ANY order (not just anbn).
  2. Solution: Use stack as a "balance tracker":
    • If 'a' arrives and B is on top: they cancel (pop B)
    • If 'a' arrives and A is on top: add to a-count (push A)
    • If 'b' arrives and A is on top: they cancel (pop A)
    • If 'b' arrives and B is on top: add to b-count (push B)
  3. Acceptance: Stack must return to Z‚ÇÄ (all balanced out).

  • Input alphabet: \(\Sigma = \{a, b, c\}\)

  • Stack alphabet: \(\Gamma = \{Z_0, a, b\}\)

Œ¥(q‚ÇÄ, a, Z‚ÇÄ) = (q‚ÇÄ, aZ‚ÇÄ)

Œ¥(q‚ÇÄ, Œµ, a) = (q‚ÇÅ, a)
Œ¥(q‚ÇÄ, Œµ, b) = (q‚ÇÅ, b)

Œ¥(q‚ÇÅ, a, a) = (q‚ÇÅ, Œµ)
Œ¥(q‚ÇÅ, b, b) = (q‚ÇÅ, Œµ)

Œ¥(q‚ÇÅ, c, Z‚ÇÄ) = (q‚ÇÇ, Z‚ÇÄ)

Œ¥(q‚ÇÇ, a, Z‚ÇÄ) = (q‚ÇÇ, Z‚ÇÄ)
Œ¥(q‚ÇÇ, b, Z‚ÇÄ) = (q‚ÇÇ, Z‚ÇÄ)

Œ¥(q‚ÇÇ, Œµ, Z‚ÇÄ) = (q‚ÇÉ, Z‚ÇÄ)
üéØ Key Strategy:
  1. Pattern: wwRcw means: string w, then its reverse, then marker 'c', then w again.
  2. Phase 1: Push first w onto stack.
  3. Phase 2: Match wR by popping (palindrome check).
  4. Phase 3: Read marker 'c'.
  5. Phase 4: Simply consume remaining w (we already verified it in phase 1!).

3.4. Summary: PDA Design PatternsÔÉÅ

üéØ Essential PDA Patterns:

1. Simple Equal Count (anbn):
‚Ä¢ Push for first symbol, pop for second
‚Ä¢ Accept when stack returns to Z‚ÇÄ

2. Palindromes (wwR):
‚Ä¢ Push first half, pop second half
‚Ä¢ Use non-determinism to guess middle

3. Independent Parts (anbmcm):
‚Ä¢ Ignore independent symbols (don't push)
‚Ä¢ Use stack only for dependent parts

4. Any-Order Equal Counts:
‚Ä¢ Push A for 'a', push B for 'b'
‚Ä¢ Pop opposite symbol when possible (cancellation)

5. Sequential Pairs (anbncmdm):
‚Ä¢ Handle first pair completely
‚Ä¢ Then handle second pair
‚Ä¢ Use different stack symbols
‚ö° PDA Limitations:
‚úì Can handle: two counts, palindromes, balanced parentheses
‚úó Cannot handle: three equal counts (anbncn), copying (ww), context-sensitive patterns
