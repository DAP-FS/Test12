2.1. Introduction

A Pushdown Automaton (PDA) is the minimal machine that can recognize languages with nested or hierarchical structure using a stack. It is the natural computational counterpart to Context-Free Grammars (CFGs) for parsing balanced patterns and matched constructs.

A PDA can accept a language either by final state or by empty stack, and both acceptance modes are equivalent in power for context-free languages.

Hint

PDAs extend finite automata with a stack memory, giving just enough power to match opens with closes (like parentheses or tags) — but still simpler than a Turing machine.

2.2. Why PDA?

  • DFA/NFA fail on nested patterns because they have only finite memory.

  • PDA succeeds by using a stack — push on every open symbol, pop on each matching close, and accept when both input and stack align.

Note

  • A PDA adds one unbounded LIFO stack to a finite automaton.

  • This lets it match recursive structures such as parentheses, expressions, or HTML tags.

2.3. Formal Definition

A PDA is a 7-tuple:

\[M = (Q, Σ, Γ, δ, q_0, Z_0, F)\]

where:

  • \(Q\) — finite set of states

  • \(Σ\) — input alphabet

  • \(Γ\) — stack alphabet

  • \(δ\) — transition relation

  • \(q_0\) — start state

  • \(Z_0\) — initial stack symbol

  • \(F\) — set of accepting states

A transition in notation:

\[δ(q, a, X) → (q', γ)\]

means: in state q, reading input symbol a (or ε), with X on top of stack — pop X, and push γ (possibly ε) while moving to state q’.

2.4. Transition Notation Explained

The format \((q, a, X) → (q', γ)\) has three parts:

Format: (current_state, input_symbol, stack_top) → (next_state, new_stack_symbols)

Example: (q₁, a, Z) → (q₁, AZ)
  → In state q₁, reading 'a', with Z on top of stack
  → Move to q₁, pop Z, push A then Z (net effect: push A)

Note: γ can be:
  • Single symbol (q, a, X) → (q', Y) [pop X, push Y]
  • Multiple symbols (q, a, X) → (q', YZ) [pop X, push Y then Z]
  • Empty (q, a, X) → (q', ε) [pop X, push nothing]

2.5. Acceptance Modes

  • Acceptance by Final State — consume entire input and end in any state in F; stack contents may remain.

  • Acceptance by Empty Stack — consume input and empty the stack; final state ignored.

Hint

Both modes recognize exactly the same class of languages: Context-Free Languages (CFLs).

2.6. PDA Example 1: Balanced Parentheses “()”

Recognize \(L = \{ w ∈ \{(,)\}^* | w \text{ is balanced} \}\)

Formal Definition:

\[M = (\{q_0, q_1, q_2\}, \{(, )\}, \{(, \$\}, δ, q_0, \$, \{q_2\})\]

Transition Rules — Format: \((q, a, X) → (q', γ)\)

Setup Transition:

(q₀, ε, ε) → (q₁, $)
Initialize stack with bottom marker ($)

Push Transitions:

(q₁, '(', ε) → (q₁, '(')
Reading '(': push '(' onto stack

Pop Transitions:

(q₁, ')', '(') → (q₁, ε)
Reading ')' with '(' on top: pop it (remove from stack)

Accept Transition:

(q₁, ε, $) → (q₂, ε)
When only $ remains, move to final state (empty stack)

2.7. PDA Example 2: Language aⁿbⁿ

Recognize \(L = \{ a^n b^n | n ≥ 1 \}\) (equal a’s followed by equal b’s).

Formal Definition:

\[M = (\{q_0, q_1, q_2, q_3\}, \{a, b\}, \{A, \$\}, δ, q_0, \$, \{q_3\})\]

Transition Rules — Format: \((q, a, X) → (q', γ)\)

Setup Transition:

(q₀, ε, ε) → (q₁, $)
Initialize stack with bottom marker ($)

Push Phase — Read a's, Push A's:

(q₁, a, ε) → (q₁, A)
Reading 'a' at stack bottom: push A
(q₁, a, A) → (q₁, AA)
Reading 'a' with A on top: push another A

Switch Phase — First b arrives, move to pop:

(q₁, b, A) → (q₂, ε)
Reading first 'b' with A on top: pop A, move to pop state

Pop Phase — Read b's, Pop A's:

(q₂, b, A) → (q₂, ε)
Reading 'b' with A on top: pop A

Accept Transition:

(q₂, ε, $) → (q₃, ε)
When only $ remains: accept (stack empty)

2.8. PDA Example 3: Palindromes WcWᴿ

Recognize \(L = \{ WcW^R | W ∈ \{a,b\}^* \}\) (palindromes with center marker ‘c’).

Formal Definition:

\[M = (\{q_0, q_1, q_2, q_3\}, \{a, b, c\}, \{a, b, \$\}, δ, q_0, \$, \{q_3\})\]

Strategy: - Push Phase (q₁) — Read W, push all symbols onto stack - Pivot Phase — Read center marker ‘c’, switch to match mode - Match Phase (q₂) — Read Wᴿ, pop must match exactly - Accept — Stack empty when input ends

Transition Rules — Format: \((q, a, X) → (q', γ)\)

Setup Transition:

(q₀, ε, ε) → (q₁, $)
Initialize stack with bottom marker ($)

Push Phase — Read W, accumulate:

(q₁, a, ε) → (q₁, a)
(q₁, a, a) → (q₁, aa)
(q₁, a, b) → (q₁, ab)
(q₁, b, ε) → (q₁, b)
(q₁, b, a) → (q₁, ba)
(q₁, b, b) → (q₁, bb)

Pivot — Read 'c', switch to match:

(q₁, c, a) → (q₂, a)
(q₁, c, b) → (q₂, b)

Match Phase — Read Wᴿ, pop matching:

(q₂, a, a) → (q₂, ε)
(q₂, b, b) → (q₂, ε)

Accept Transition:

(q₂, ε, $) → (q₃, ε)

2.9. PDA Example 4: Equal Count aⁿbⁿ (Any Order)

Recognize \(L = \{ w ∈ \{a,b\}^* | \#_a(w) = \#_b(w) \}\) (equal a’s and b’s in any order).

Formal Definition:

\[M = (\{q_0, q_1, q_2\}, \{a, b\}, \{A, B, \$\}, δ, q_0, \$, \{q_2\})\]

Strategy: - Push A when ‘a’ exceeds ‘b’ - Push B when ‘b’ exceeds ‘a’ - Pop when opposite symbol arrives (cancellation) - Accept when stack returns to $ only

Transition Rules — Format: \((q, a, X) → (q', γ)\)

Setup Transition:

(q₀, ε, ε) → (q₁, $)
Initialize stack with bottom marker ($)

Read 'a' — Push A or Pop B:

(q₁, a, $) → (q₁, A$)
First 'a' with $ bottom: push A
(q₁, a, A) → (q₁, AA)
'a' with A on top: push another A (increase a-count)
(q₁, a, B) → (q₁, ε)
'a' with B on top: pop B (cancel b with a)

Read 'b' — Push B or Pop A:

(q₁, b, $) → (q₁, B$)
First 'b' with $ bottom: push B
(q₁, b, B) → (q₁, BB)
'b' with B on top: push another B (increase b-count)
(q₁, b, A) → (q₁, ε)
'b' with A on top: pop A (cancel a with b)

Accept Transition:

(q₁, ε, $) → (q₂, ε)
When only $ remains: accept (equal counts)

2.11. Key Principles for Reading Transitions

🎯 How to Understand (q, a, X) → (q', γ) (Click to Expand)

Component 1: (q, a, X)
• q = current state
• a = input symbol being read (or ε for no input consumed)
• X = stack top symbol being popped

Component 2: (q', γ)
• q' = next state to move to
• γ = string to push (can be empty ε, single symbol, or multiple)

Order of Stack Operations:
1. POP: Remove X from stack top
2. PUSH: Add symbols in γ left-to-right (leftmost goes first)

Examples:
• (q, a, X) → (q, ε) = consume 'a', pop X, push nothing
• (q, a, X) → (q, YZ) = consume 'a', pop X, push Y then Z (Y on top)
• (q, ε, X) → (q, ε) = no input, pop X, push nothing (spontaneous move)

2.12. Summary

  • Transition notation \((q, a, X) → (q', γ)\) specifies exact stack operations

  • Push phase accumulates symbols (stack grows)

  • Pop phase matches and removes symbols (stack shrinks)

  • Cancellation allows balancing in any order

  • Combined visualizations show both stack evolution and formal transitions

  • Parallel diagrams make acceptance/rejection immediately visible

  • Foundation for parser design and syntax analysis
